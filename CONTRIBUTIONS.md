# 📥 CONTRIBUTIONS

Merci de votre intérêt pour contribuer à **NoteForgeDev** !
Ce projet open source a pour objectif de devenir une bibliothèque riche en **notes de cours** pour apprendre le software engineering par la lecture.

## 💡 Comment contribuer

Vous pouvez contribuer de plusieurs façons :

### ✏️ Proposer une nouvelle leçon

1. Choisissez le dossier approprié (`/cours/javascript`, `/cours/php`, etc.).
2. Créez un fichier Markdown au format `XX-titre.md` (par ex. `03-boucles.md`), où `XX` est le numéro de l’ordre.
3. Respectez le squelette de leçon fourni (voir le modèle dans chaque dossier).
4. Vérifiez l’orthographe et la clarté.
5. Faites une **pull request (PR)** sur GitHub.

---

### 🛠️ Améliorer une leçon existante

- Corrigez des fautes d’orthographe ou de grammaire.
- Ajoutez des explications plus claires.
- Proposez des exercices ou des liens complémentaires.

---

### 🌍 Traduire des leçons

1. Allez dans le dossier `/traductions/en` pour l’anglais.
2. Traduisez le fichier correspondant en respectant le style et la structure.
3. Faites une pull request.

---

### 💬 Signaler un problème

- Ouvrez une **issue** sur GitHub si vous trouvez :
  - une erreur,
  - un contenu peu clair,
  - une section manquante.

---

## 📐 Bonnes pratiques

✅ Respectez la langue principale du projet (français) pour les contributions principales.
✅ Utilisez un langage simple, clair et pédagogique.
✅ Citez vos sources si vous ajoutez des contenus externes.
✅ Testez les exemples de code avant de les soumettre.
